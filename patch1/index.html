═══════════════════════════════════════════════════════════════

Detect platform first, then pick the right path:

──────────────────  Windows  ──────────────────

The pattern is .lnk → .bat → claude. You may already have a working .bat in your AI folder (like Joyce does). If not, you'll create one. Then create a Desktop shortcut that points to it with the custom icon I drew.

Run this whole block in PowerShell (open Windows Terminal → PowerShell tab):

    # ── Adjust these three to your real values from your CLAUDE.md ──
    $AiName    = ""      # e.g. Sherlock
    $VaultName = ""   # e.g. Mindcastle
    $Slug      = ""         # e.g. sherlock (lowercase, dashes for spaces)

    # ── Resolve real paths (handles OneDrive Desktop redirection) ──
    $Desktop   = [Environment]::GetFolderPath('Desktop')
    $VaultPath = Join-Path (Join-Path $Desktop $AiName) $VaultName
    $BatPath   = Join-Path $VaultPath "$AiName.bat"
    $IconDir   = Join-Path $VaultPath ".icon"
    $IconPng   = Join-Path $IconDir   "$Slug.png"
    $IconIco   = Join-Path $IconDir   "$Slug.ico"
    $LinkPath  = Join-Path $Desktop   "$AiName.lnk"

    New-Item -ItemType Directory -Force -Path $IconDir | Out-Null

    # ── Create the .bat if you don't already have one ──
    if (-not (Test-Path $BatPath)) {
      @"
    @echo off
    cd /d `"$VaultPath`"
    claude --dangerously-skip-permissions
    "@ | Set-Content -Path $BatPath -Encoding ASCII
    }

    # ── Pull the icon ──
    if (-not (Test-Path $IconPng)) {
      Invoke-WebRequest -Uri "https://stevenlava.com/icons/$Slug.png" -OutFile $IconPng
    }

    # ── Convert .png → .ico (built-in via System.Drawing) ──
    Add-Type -AssemblyName System.Drawing
    $bmp  = [System.Drawing.Image]::FromFile($IconPng)
    $icon = [System.Drawing.Icon]::FromHandle($bmp.GetHicon())
    $fs   = [System.IO.File]::OpenWrite($IconIco)
    $icon.Save($fs); $fs.Close(); $bmp.Dispose()

    # ── Build the Desktop shortcut pointing at the .bat ──
    $WshShell = New-Object -ComObject WScript.Shell
    $Shortcut = $WshShell.CreateShortcut($LinkPath)
    $Shortcut.TargetPath       = $BatPath
    $Shortcut.WorkingDirectory = $VaultPath
    $Shortcut.IconLocation     = "$IconIco,0"
    $Shortcut.Save()

    # ── Refresh the icon cache so the new icon shows immediately ──
    Stop-Process -Name explorer -Force; Start-Process explorer

    Write-Host "Launcher ready: $LinkPath"

Why each piece:
  - [Environment]::GetFolderPath('Desktop')  →  resolves to OneDrive\Desktop on Win11 with KFM, plain Desktop otherwise. Hardcoding $env:USERPROFILE\Desktop puts the .lnk in the wrong folder if OneDrive sync is on.
  - .lnk → .bat → claude  →  the .bat does the cd /d into the vault so Claude opens with full project context. The shortcut just gets to be the pretty icon.
  - System.Drawing PNG→ICO  →  no extra tools needed, ships with .NET.
  - Stop-Process explorer  →  Windows caches icons aggressively; restarting Explorer is the fastest way to see the new icon on the new shortcut.

──────────────────  macOS  ──────────────────

If ~/Desktop/.app exists with a proper icon, you're done — confirm and skip. Otherwise rebuild:

    APP="$HOME/Desktop/.app"
    SLUG=""
    VAULT=""
    ICON_PNG="$HOME/Desktop//$VAULT/.icon/$SLUG.png"

    mkdir -p "$(dirname "$ICON_PNG")"
    [ -f "$ICON_PNG" ] || curl -sSL "https://stevenlava.com/icons/$SLUG.png" -o "$ICON_PNG"

    ICONSET="/tmp/$SLUG.iconset"
    rm -rf "$ICONSET"; mkdir -p "$ICONSET"
    for size in 16 32 64 128 256 512; do
      sips -z $size $size "$ICON_PNG" --out "$ICONSET/icon_${size}x${size}.png" >/dev/null
      sips -z $((size*2)) $((size*2)) "$ICON_PNG" --out "$ICONSET/icon_${size}x${size}@2x.png" >/dev/null
    done
    iconutil -c icns "$ICONSET" -o "/tmp/$SLUG.icns"

    SCRIPT="tell application \"Terminal\" to do script \"cd ~/Desktop//$VAULT && claude --dangerously-skip-permissions\""
    osacompile -o "$APP" -e "$SCRIPT"
    cp "/tmp/$SLUG.icns" "$APP/Contents/Resources/applet.icns"
    touch "$APP"
    killall Finder
