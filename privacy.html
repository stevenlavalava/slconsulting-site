Privacy Policy

Last updated: April 22, 2026

This privacy policy describes how Steven Lava (“I,” “my,” or “me”) handles data accessed by personal productivity software used in the course of operational consulting work at stevenlava.com.

Who This Applies To

This policy applies to a personal-use automation system (“Bilbo”) operated by me to assist with client engagements, personal task management, and meeting workflow for SL Consulting. The system is not a product sold to or shared with third parties.

What Data Is Accessed

When I authenticate the system with my Google account, the following scopes are requested via OAuth:

  • Google Sheets — read and write access to spreadsheets I own or have been shared with me, used to maintain weekly client scorecards and meeting data.
  • Google Drive — read and write access to files I own, used to store working documents and meeting artifacts.
  • Gmail (read & send) — read access used as a secondary channel for retrieving email attachments; send access used as a backup method for outbound email.
  • Google Calendar — event read and write access for scheduling meetings and personal commitments.
  • Google Apps Script — project and deployment management, used to deploy Apps Script endpoints that support my own workflow tooling.

How Data Is Used

All data access occurs only in the context of my own personal workflow. The system reads and writes only to accounts, sheets, and folders that I own or have been explicitly shared on. No data is transmitted to any third party, uploaded to any external service other than Google’s own APIs, or used for any training or analytics.

How Data Is Stored

OAuth tokens are stored locally on my devices (MacBook and desk computer), encrypted at rest via macOS file system protections. Tokens are never shared, never committed to version control, and never transmitted over any channel other than Google’s official OAuth endpoints.

No user data beyond OAuth tokens is persisted by the automation system. The system reads data on demand and writes output back to the same Google services. There is no separate database, no analytics store, and no third-party logging.

Sharing

I do not share, sell, or disclose Google user data to any third party. The system accesses my own Google account only, on my own devices.

Retention

OAuth refresh tokens are retained locally until revoked. I can revoke access at any time via my Google Account security settings, which immediately invalidates the stored tokens.

Security

The system uses Google’s standard OAuth 2.0 flow. No passwords are stored. Tokens are kept on local disk only, accessible only to my user account on each device. Devices are protected with FileVault full-disk encryption, Touch ID / Face ID login, and macOS automatic security updates.
